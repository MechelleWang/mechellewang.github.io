GraphQL是什么

GraphQL是一种服务器检索数据的查询语言，和RESTful API、SOAP、gRPC的作用是一样的，但比这些都更加灵活和智能。比如当修改一个接口的URL上的请求参数时，RESTful 需要重新定义、增删改接口，而GraghQL接口无需做任何后端修改。再比如，当前端要求接口只返回需要的字段时，后端只能通过定义各种各样的Model来支持，而GraphQL接口无需做过多的修改，就能够准确地指定响应中返回的数据，不多也不少。再比如，当使用接口获取特定艺术家所有专辑所有曲目的所有标题的列表，则需要发出多个请求，首先查询艺术家ID，然后查找所有唱片集，最后查找每个唱片集上的所有曲目，这些步骤在GraphQL中可以通过一个查询直接访问到所有数据。

Java8 Locks

Posted on 2020-08-26 Edited on 2020-09-07

在Java中，多个线程对共享资源进行访问，可以通过synchronized和锁来实现。synchronized无需显示地获取和释放锁，无需关注锁的释放顺序，当方法或代码块执行结束锁就自动释放了，这样做的好处在于一定程度上减少编程错误。但这样也导致了synchronized不够灵活，有些情况无法满足需求：

  • 需要实现一个公平锁，locks包下的ReentrantLock支持公平锁的实现
  • 减少未获得锁的线程的阻塞时间，Lock API中可以通过tryLock()方法减少阻塞时间
  • 终止处于等待状态的线程，Lock API提供了lockInterruptibly()方法终止处于等待状态的线程
  • 需要显示的指定锁的释放和获取顺序

java.util.concurrent.locks包中支持各种各样的基于Lock接口等实现的锁，相比synchronized，锁提供了更灵活的控制操作。这篇博文主要介绍Java8中常用的锁的类型和使用场景。下面这幅图是java.util.concurrent.locks包下常用锁接口及其子类的关系。

Java Generics

Posted on 2020-08-21 Edited on 2020-09-08

Java泛型是一种语言特性，允许定义和使用泛型类型和方法。泛型类型或方法与常规的Java类型和方法的不同之处在于泛型类型具有类型参数。

为什么要使用泛型

我们在写代码的时候经常希望一种集合中只包含某种类型的元素，比如包含Integer/String的列表。在原始集合框架中，如果不在集合中做额外的检查则不能拥有同类型的集合。泛型的引入很好的解决了这一类问题，它帮助我们在编译时自动在代码中添加类型检查，只有编译正确的代码才能进入运行，而在运行时则自动擦除类型。

通俗地说，泛型强制Java语言中的类型安全性。

如果没有这种类型安全性的保证，代码中可能会出现各种各样的BUG，并且这些BUG只能在运行时暴露，大大降低了代码的健壮性和编写效率。

泛型通过在编译时检测到更多的bug来增加代码的稳定性。

第一步我们了解了在Java中为什么要使用泛型，第二步需要了解泛型如何在Java中生效。

泛型在Java中如何生效

泛型有两大特性，一是类型安全，保证编译时期的类型安全，防止引起不必要的ClassCastException。二是类型擦除，从本质上讲，使用泛型向源代码中添加的任何额外信息都会在字节码生成之前删除。也就是说字节码中使用的是旧的Java代码，这样做的好处是完美的兼容了Java5之前编写的代码。
